ПРАВИЛА РАБОТЫ НАД ПРОЕКТОМ

Ветки (branches)

- `main` — только абсолютно рабочая версия. Сюда ничего не пушим напрямую!
- `feature/название` — для новых фич. Например: `feature/add-enemy`
- `bugfix/название` — для исправления ошибок. Например: `bugfix/fix-crash`

Процесс работы

1. Начинаем новую задачу
- В GitHub Desktop переключаемся на ветку `main`
- Жмём Branch → New branch
- Называем ветку по правилам выше
- Жмём "Create branch", затем "Publish branch"

2. Работаем
- Делаем изменения в Godot
- Сохраняем
- В GitHub Desktop пишем Summary (что сделали)
- Жмём "Commit to ..."
- Жмём "Push origin"

3. Предлагаем изменения в main
- В GitHub Desktop после пуша жмём "Create Pull Request"
- Или заходим на GitHub → жёлтая плашка → "Compare & pull request"
- Проверяем: `base: main` ⬅ `compare: наша-ветка`
- Пишем понятное описание
- Жмём "Create pull request"

4. Ждём проверок
GitHub Actions автоматически соберёт игру. В PR (pull request) будет видно:
- зелёная галочка — всё ок, можно мёржить
- красный крест — что-то сломалось, чиним

### 5. Мержим
- Когда всё зелёное, жмём "Merge pull request** → "Confirm merge"
- Ветку можно удалить (кнопка "Delete branch")

## ✅ Важные правила

1. Никогда не пушить в main напрямую! Только через Pull Request.
2. Всегда делать Pull (Fetch origin) перед началом работы, чтобы забрать чужие изменения.
3. Писать понятные комментарии в коммитах. Не "фыва", а "Добавил спрайт игрока".
4. Не коммитить служебные папки (`.godot`, `.import`) — `.gitignore` сам их не пустит.
5. Если что-то пошло не так — пишите в чат, разберёмся.

## 🎮 Где скачать готовую игру
Все сборки автоматически выкладываются на вкладке **Actions**. Заходите → выбираете последний запуск → **Artifacts**.
